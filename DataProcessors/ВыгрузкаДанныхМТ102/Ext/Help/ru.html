Выгрузка данных в формате МТ-102

Обработка предназначена для формирования файлов выгрузки в формате МТ-102 для:

  • Перечисления заработной платы на карт-счета работников в банк;
  • Перечисления обязательных пенсионных взносов и обязательных профессиональных пенсионных взносов в накопительные пенсионные фонды;
  • Перечисления обязательных социальных отчислений, взносов и отчислений на ОСМС.

При формировании swift-файлов доступны следующие настройки:

Для всех видов перечислений:

  • Формировать отдельные транспортные файлы при превышении 500 записей;
  • Включать поле «FT» всегда (даже при отсутствии отчества у сотрудника).
  • Включать поля формата ISO 20022 – при включении в формат выгрузки Swift-файла (МТ-102), для поддержки перехода на стандарт ISO 20022, добавляются следующие дополнительные поля:

В заголовке файла:

• Наименование и ИИН/БИН Фактического плательщика, при его наличии в документе, в новый раздел «:80:»;
• Данные отправителя дополняются кодом страны резиденства /CTRYOFRES/ (для юридических лиц - страна регистрации, для физических лиц - страна гражданства);
• Данные бенефициара также дополняются кодом страны резиденства /CTRYOFRES/.

В списке работников в зависимости от вида перечисления:

• Признак резидентства в поле /IRS/;
• Сектор экономики в поле /SECO/;
• Код страны резиденства (только для нерезидентов) /CTRYOFRES/.

Устанавливать значение данной настройки необходимо после даты ввода действия постановления Правления Национального Банка РК №105 от 25.12.2023 г., если программное обеспечение банка поддерживает новые поля формата ISO 20022 в МТ-102 (данную информацию можно получить в обслуживающем банке).

Для перечисления обязательных пенсионных взносов, обязательных профессиональных пенсионных взносов и обязательных социальных отчислений:

  • /PSO – признак специальной обработки. При включении признака Включать поле PSO поле включается в файл со значением «01».
  • /PRT – приоритет документа. При включении признака Включать поле PRT поле включается в файл со значением «05». Если ключевое поле в сообщении отсутствует, при обработке документа в банковских системах документу по умолчанию присваивается приоритет 50.

Для перечисления обязательных социальных отчислений, отчислений на обязательное социальное медицинское страхование, взносов на обязательное социальное медицинское страхование:

  • Поле «PERIOD» в заголовке. При включении признака Поле PERIOD в заголовке поле PERIOD будет указываться в заголовке файла. Применяется в случае перечисления социальных отчислений за один период.
  • Поле «PERIOD» в списке. При включении признака Поле PERIOD в списке поле PERIOD будет указываться для каждого сотрудника. Применяется в случае перечисления социальных отчислений за несколько периодов.

«Разделять суммы по периоду задолженности». При включении признака Разделять суммы по периоду задолженности - период указывается в каждой строке swift файла, при этом суммы делятся по периодам согласно табличной части документа Ведомость на перечисление взносов, отчислений. Применяется в случае перечисления социальных отчислений, отчислений на обязательное социальное медицинское страхование, обязательных пенсионных взносов, взносов на обязательное социальное медицинское страхование.
Настройка доступна, если установлен признак Поле PERIOD в списке.
